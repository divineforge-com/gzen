## 2024-05-18 - [Dropdown Keyboard Accessibility]
**Learning:** Native CSS `:focus-within` on custom dropdown components often relies heavily on mouse interactions like `:hover` and `tabindex` flows that dont inherently update state variables like `aria-expanded` attributes for screen readers. Using `aria-expanded` and explicit space/enter toggle actions alongside native JS logic is necessary to make the header navigation keyboard navigable for assistive tech.
**Action:** When creating or modifying dropdown navigational components in this app, ensure a JavaScript listener is present to toggle `aria-expanded` states via keyboard events (Space/Enter), handle `Escape` to close, and pair this explicitly with CSS that triggers the dropdown display on `[aria-expanded="true"]`.
