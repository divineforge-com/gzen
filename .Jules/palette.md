## 2026-07-04 - Multilingual Link Accessibility
**Learning:** The application's language switcher script in `static/js/language-toggle.js` targets the specific ID `#lang-toggle` and uses `data-lang` attributes. Setting the `hreflang` and `lang` attributes natively within the template (e.g. `layouts/partials/header/basic.html`) is critical not only for accessibility (allowing screen readers to switch pronunciation profiles) but also perfectly aligns with how Hugo handles `.Language.Lang` iteration naturally.
**Action:** Always ensure that dynamically generated language switcher links output explicit `hreflang`, `lang`, and `data-lang` attributes mapped to `.Language.Lang` when implementing multilingual sites, and verify their target container ID correctly matches any associated JS handlers.
